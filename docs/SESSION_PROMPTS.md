# Промпты Михаила для сессий Claude Code

Копируй как есть. Стартовый — первым сообщением новой сессии; завершающий — последним,
когда работа закончена. Оба нарочно короткие: детали живут в HANDOVER/FIX_JOURNAL/памяти,
а не в промпте (промпт не должен устаревать).

---

## 🚀 Старт сессии

```
Это проект КРЕСТ. Перед работой сделай по порядку:

1. Прочитай docs/HANDOVER.md — точка входа: открытые задачи, ключевой контекст,
   что уже сделано (НЕ переделывай сделанное).
2. Прочитай верх docs/FIX_JOURNAL.md (2-3 верхние даты) — хронология правок и багов,
   чтобы не повторять и не «изобретать» уже сделанное.
3. Память проекта (memory/MEMORY.md) грузится сама — ориентируйся по индексу; свежая
   сводка = последний session_*-файл в индексе. Отдельные файлы памяти открывай,
   только если тема совпала.
4. Пробеги: git log --oneline -20 — последние коммиты и реальное состояние.
5. CLAUDE.md и .claude/rules/* уже в контексте — соблюдай их.

Правила:
- Отвечай ТОЛЬКО на русском.
- НЕ галлюцинируй: не знаешь текущее состояние — проверь код/БД/git, не придумывай.
  Прежде чем «чинить» — убедись, что оно уже не работает.
- Перед крупной правкой критичного пути (онбординг, дневная модель, гейт блоков,
  роли/scope панели) — сверься с FIX_JOURNAL и подтверди план.
- Проверка перед коммитом: cd apps/web && npx tsc --noEmit (+ npm --workspace=@krest/web
  run build для важного). Коммить и пушь в main (Vercel автодеплой).
- Прод-данные меняй осторожно и только по явной просьбе; SQL — через Supabase MCP;
  схема — только миграциями в supabase/migrations/.
- Таймзоны: per-student даты — только через studentLocalToday (пояс города), не UTC/Бали.
- Найдёшь по пути баг/дыру вне задачи — не чини молча: скажи и предложи.

Прочитал — подтверди одной строкой, что ты в контексте, и жди задачу.
```

## 🏁 Завершение сессии

```
Закрываем сессию КРЕСТ. Сделай по порядку, компактно:

1. docs/FIX_JOURNAL.md — добавь СВЕРХУ раздел с сегодняшней датой: что сделано,
   найденные/починенные баги (причина → фикс → коммит), что предлагалось и было
   ОТКЛОНЕНО (чтобы не предлагали заново). Обнови строку «ОТКРЫТО (на дату)» в шапке.
2. docs/HANDOVER.md — обнови дату и «Открытые задачи»: что закрыто, что осталось,
   что появилось; в «Ключевой контекст» — новые правила/каноны/прод-данные.
3. Память — только то, чего НЕТ в git: короткий memory/session_ГГГГ_ММ_ДД_*.md
   (решения Михаила, договорённости, изменённые прод-данные, грабли) + одна строка
   в MEMORY.md. Устаревшие заметки памяти поправь или удали.
4. Проверь чистоту: git status без незакоммиченного, tsc/build зелёные, всё запушено.
   Журнальные правки закоммить отдельным docs-коммитом.
5. Ответь финальной сводкой: сделано (с хешами коммитов) · открыто · чем продолжить
   следующую сессию (1-2 строки).

Не раздувай файлы: журнал — по делу, память — краткая, без пересказа кода.
```
