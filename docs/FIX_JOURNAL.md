### ИИ засчитывал случайные фото вместо креста (палетка теней → ✅)
- **Причина:** роут `cross-photo/upload` намеренно НЕ блокировал при `matched=false` («мягко, записываем всегда» — раньше так сделали, чтобы 422 не резал настоящие кресты при баге «загрузка не регистрировалась»). Из-за этого любое фото проходило с ✅.
- **Фикс:** вернул блокировку явного «не креста» — `if (aiMatched === false) return 422 NOT_A_CROSS` (фото не сохраняется, день не закрывается). Блокирует ТОЛЬКО явный `false`; ошибка ИИ / HEIC / accel / сбой парсинга = fail-open (не режет настоящие). Промпт `check.ts` усилен: сфотографированный предмет/косметика/скриншот/селфи = `matched=false` всегда.
- **Плюс:** на экране фото — удаление фото ПОСЛЕДНЕГО дня (`/api/m/cross-photo/delete`, только самый свежий день, чтобы не «раскрыть» прошлый закрытый; inline-подтверждение).

### % курса показывал 0% при 5/7 закрытых днях блока 1
- **Причина:** `coursePct = сданныеБлоки / всего` (только блоки с `block_passed_at`). Пока идёт блок 1 (не сдан) → 0/10 = 0%, дни внутри блока не учитывались.
- **Фикс (`BlockList`, коммит f7c36cc):** `pct = (passed + min(closedDaysТекущегоБлока/7, 1)) / total`. 5/7 дней блока 1 → ~7%. Не баг данных — формула была «только сданные блоки».

### Лента — кружок не отправлялся («Не удалось отправить») на iPhone
- **Причина (главная):** на iOS mime записи = `video/mp4;codecs=avc1`, а бакет `community-media` в `allowed_mime_types` разрешает строго `video/mp4` → Supabase отклонял upload. Роут публикации брал `file.type` как есть, не отбрасывая `;codecs`.
- **Фикс:** в `api/m/community/post` mime чистится `.split(';')[0]` (как в роуте местописаний).
- **Сопутствующее:** битрейт видео-кружка снижен (360к видео + 48к аудио ≈ 3 МБ/60с) — раньше после фикса зависания полные 60с при 500к давали ~4+ МБ и риск упереться в лимит тела Vercel 4.5 МБ; + клиентский гард с понятным текстом, если запись всё же >4.3 МБ. Те же настройки (fps-кап, `start(1000)`, битрейт) продублированы в видеокружок местописаний (`LocationCard`).

- **Область:** `api/m/prayer/[blockId]/route.ts`, `m/prayer/[blockId]/PrayerClient.tsx`.
