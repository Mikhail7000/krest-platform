## 1. Открытие блока

Ученик заходит в Блок 1 «Малый Крест». Видит сразу **12 карточек-пунктов ДЗ** (см. секцию 3) и **ресурсы блока** в боковом или скрываемом разделе.

Состояния каждой карточки:
- 🔒 `locked` — недоступно (предыдущий шаг не завершён, если применимо)
- 🆕 `available` — можно начать
- ⏳ `in_progress` — есть submission в pending review
- ✅ `approved` — одобрено куратором
- 🔄 `rejected` — отклонено, нужна переделка
- ✅ `auto_approved` — авто-засчитано (просмотр видео)

## 3. Домашнее задание — 12 пунктов

Структура одинакова для всех 10 блоков. Реализуется через таблицу `assignments` (12 строк на каждый блок) и `submissions` (фактические сдачи ученика).

| # | Пункт | Тип | Обязательно | Workflow куратора |
|---|---|---|---|---|
| 1 | Подготовка | info-экран | — | авто |
| 2 | Просмотр основного видео | auto (no-skip ≥95%) | ✅ | авто |
| 3 | Дополнительное видео | auto (no-skip ≥95%) | ✅ | авто |
| 4 | Форум-рефлексия (3 вопроса) | text | ✅ | одобрить / вернуть / коммент |
| 5 | Конспект | text/photo | ✅ | одобрить / вернуть / коммент |
| 6 | Писать крест ежедневно | photo daily (мин 7 дней) | ✅ | одобрить день / коммент |
| 7 | Местописания (видео) | video-кружок ИЛИ загрузка | ✅ | одобрить / вернуть / коммент |
| 8 | Прослушать молитвы (только Блок 1) | auto | — | авто |
| 9 | Молитва ежедневно | галочка-чекин daily | — | просмотр (на доверии) |
| 10 | Сдача блока куратору | manual approve (офлайн) | ✅ block exam | одобрить «Сдал» / 🔄 «Иди ещё раз» |
| 11 | Эпоха пятницы (практика) | text + photo + voice | ✅ для всех (включая удалённых) | одобрить / коммент |
| 12 | Эмоции + ежедневный отчёт | text daily | ✅ | просмотр; **алерт куратору при пропуске** |

### Детали ключевых пунктов

**Пункт 4 — Форум-рефлексия.** 3 поля по ≥50 символов: «Что узнал нового», «Что вдохновило», «Что беру в практику». При submit → push куратору с превью текста.

**Пункт 5 — Конспект.** Загрузка одного из: текст ≥200 символов ИЛИ фото рукописного конспекта. Куратор одобряет / возвращает с конкретными замечаниями.

**Пункт 6 — Писать крест ежедневно.** Каждый день фото переписанного текста. Счётчик «N из 7 дней». Block gate требует **минимум 7 уникальных дней** одобренных submission. Путь хранения: `student-submissions/{user_id}/block-1/photos/`.

**Пункт 7 — Местописания.** В UI две кнопки:
- 🔴 **Записать кругляшок** — нативно через `MediaRecorder.start()`, single-take, max 60 сек
- 📁 **Загрузить видео** — обычная загрузка из галереи (mp4/mov, до 100 МБ)

Запись с камеры через `getUserMedia({ video: true, audio: true })`. **input file для камеры не используется** (нет «выбора готового видео»). Хранение: `student-submissions/{user_id}/block-1/scripture-videos/`.

**Пункт 9 — Молитва.** Только галочка «помолился сегодня — да» каждый день. Без доказательств, на доверии. Куратор видит дисциплину в дневном календаре.

**Пункт 10 — Сдача куратору (block exam).** Кнопка «Готов сдавать» активна только когда все остальные обязательные ✅-пункты одобрены. При нажатии → push куратору «[имя] просит сдать Блок 1, договорись о горнице».

**Сдача только офлайн.** На Бали — устный пересказ на горнице. В других городах — куратор сам решает как принять (вне платформы). Видеосозвон в платформе **не реализован**.

После очной встречи куратор в админке ставит ✅ «Сдал» (с опц. комментарием) или 🔄 «Иди ещё раз» (с обязательным комментарием).

**Пункт 11 — Эпоха пятницы.** Обязательна для всех учеников (включая удалённых от Бали). Submission: текст + фото + voice кружок (опционально, какой удобнее).

**Пункт 12 — Ежедневный отчёт.** Ежедневная короткая запись свободной формы. При submit → push куратору с содержимым. **Если день закончился без отчёта** (cron 23:59 в timezone города ученика) → алерт куратору: «⚠️ [имя] не отчиталась 1 день». 3+ дня тишины → эскалация: «🚨 [имя] 3 дня тишины — напомни».

## 4. Иерархия гейтов (block / mid / final)

### Block gate (после каждого блока)

```sql
SELECT is_block_completed(user_id, block_id);
-- TRUE когда все обязательные ✅-пункты одобрены
-- Для recurring пунктов (6, 12) — минимум 7 уникальных дней
```

При TRUE → триггер `trigger_unlock_next_block` создаёт запись `course_progress` для следующего блока + push ученику: «✅ Блок [N] пройден! Открыт [N+1]: [название]».

### Mid-exam (только один — после Блока 5)

После approval пункта 10 Блока 5 «Состояние Неверующего» создаётся `exam` типа `mid` со статусом `pending`. **Блок 6 заблокирован** до сдачи mid-exam.

Принимающий куратор — **другой**, не свой (валидация в API: `exam.examiner_id != student.curator_id`). Сдача офлайн. При passing → разблокируется Блок 6.

### Final exam (после Блока 10)

После approval пункта 10 Блока 10 «5 Уверенностей» создаётся `exam` типа `final` со статусом `pending`. Принимающий — `admin` или `super_admin`.

При passing:
- `course_progress[course_id=1, user_id].status = 'completed'`
- Триггер `trigger_unlock_next_course` создаёт `course_progress` для курса «10 писем» (если он `active`) или `pending_course_activation` (если `coming_soon`)
- Анимация ачивки в UI:
  ```
  🏆 Поздравляем! Теперь вы Мастер Креста.
  Поздравляем, возможно, ты уже юноша.
  [1 Иоанна 2:14]
  ```
- Видимость ученика расширяется: теперь он видит **всех учеников платформы** (через функцию `is_visible_to`).

## 5. Роли и видимость

### Иерархия

| Роль | Назначает | Что может |
|---|---|---|
| `super_admin` | По seed (Михаил, Алекс, Эля, Игорь) | Всё. Назначать любые роли. CRUD городов. Передача управления |
| `admin` | super_admin | Назначать кураторов в зоне, прикреплять учеников |
| `curator` | super_admin / admin | Принимать ДЗ. Видеть свою группу + кураторов своего города |
| `student` | super_admin / admin / curator | Учиться |

«Учеников брать себе» могут все три верхние роли. Прикрепление ученика к конкретному куратору — через `/api/admin/student/attach-curator` (super_admin / admin).

### Видимость по прогрессии

- Студент пока учится КРЕСТ → видит только свою группу (`profiles.curator_id = my_curator`)
- Сдал КРЕСТ (final exam) → видит **всех учеников платформы** глобально
- Куратор видит: своих учеников + других кураторов своего города + (опционально) их учеников
- Admin / super-admin видят всё

Реализация — PL/pgSQL `is_visible_to(viewer_id, target_id)` + RLS на `profiles`, `course_progress`, `daily_activity`.

### Дневной трекинг куратора

Куратор видит **календарную таблицу** своей группы (см. SPEC v3.0 US-004): ученики × дни недели, в каждой ячейке иконки активности (отчёт, фото креста, кругляшок). Сверху — алерт-список «3+ дня тишины».

Реализация: таблица `daily_activity` (одна запись на user × log_date) + cron `silence-check` каждый час в timezone городов.

## 6. Workflow куратора по submission

| Действие | Эффект |
|---|---|
| ✅ Approve (одобрить) | submission.status = 'approved', user видит ✅ в карточке пункта |
| 🔄 Reject (вернуть) | submission.status = 'rejected'; reviewer_comment ≥10 символов обязателен; ученик может перезалить |
| 💬 Approve с комментарием | approved + опциональный текст для ученика |
| 📈 Просмотр без действия | для пунктов 9 (молитва), 12 (отчёт) — куратор только видит |

Все действия → push в Telegram ученику + запись в `notifications_log`. История версий submission сохраняется (новая запись, старая помечается obsolete).

## 8. Чат ученик ↔ куратор

- Каждый ученик имеет **встроенный чат со своим куратором** (`profiles.curator_id`)
- Доступен из MiniApp `/m/chat` и веб-сайта
- Реалтайм через Supabase Realtime (subscribe на `direct_messages`)
- Push в Telegram при новом сообщении
- Vложения: текст + voice (через MediaRecorder, max 60 сек) + опц. фото

Куратор может писать всем своим ученикам. Ученик — только своему куратору.

## 9. ИИ-тренажёр стихов

Отдельный модуль `/m/trainer`. Доступны только стихи **уже завершённых блоков** (gate: тренажёр блока N+1 заблокирован пока N не сдан).

- UI: показать референс («Иоанна 19:30») → ученик пишет наизусть
- ИИ через Anthropic Messages API (claude-sonnet-4-6) проверяет:
  - `approved` — точное совпадение, мелкие опечатки до 2 символов
  - `approved_with_notes` — небольшая перефразировка с сохранением смысла
  - `rejected` — существенное искажение, пропуск ключевых слов
- Diff: «правильно: ...», «у тебя: ...»
- Прогресс: 3 успешных подряд → стих помечается `mastered`

**Fallback** при недоступности Anthropic — Levenshtein distance ≤2.

**Бюджет:** ~$0.003 per request, ~$90/мес на 100 учеников.
