## Что это и зачем

Обратный инжиниринг — это системный анализ чужого продукта с целью понять:
- Как он устроен технически (стек, архитектура, интеграции)
- Какие UX-паттерны использует и почему они работают
- Какая бизнес-модель стоит за продуктом
- Где его сильные стороны, а где — уязвимости
- Что можно взять как референс для своего проекта

Это **первый шаг** перед созданием любого продукта. Прежде чем писать спецификацию — изучи тех, кто уже решает похожую проблему.

## СТРУКТУРА АНАЛИЗА

### УРОВЕНЬ 1: Структурный анализ
Проанализируй архитектуру продукта:
- Общая навигационная структура (карта страниц/экранов)
- Иерархия информации — что показывается первым, что скрыто
- Потоки пользователя (user flows) — от входа до целевого действия
- Модульность — как связаны части продукта между собой
- Количество кликов до ключевого действия (time-to-value)

### УРОВЕНЬ 2: Технический стек
Определи технологии (по признакам в коде, структуре URL, meta-тегам, поведению):
- Frontend: фреймворк (React/Next.js/Vue/etc.), CSS-решение, UI-библиотека
- Backend: серверная платформа (Node.js/Python/etc.), API-архитектура (REST/GraphQL)
- База данных: тип (SQL/NoSQL), предположения по схеме данных
- Хостинг: где развернут (Vercel/AWS/VPS), CDN, домен
- Интеграции: платежи, аналитика, чат-виджеты, email-сервисы
- Аутентификация: способ входа (email/OAuth/magic link)
Если не можешь определить точно — укажи 2-3 наиболее вероятных варианта с обоснованием.

### УРОВЕНЬ 3: UX/UI анализ
- Дизайн-система: типографика, цветовая палитра, spacing, иконки
- Паттерны взаимодействия: формы, модальные окна, анимации, микро-интеракции
- Адаптивность: как продукт выглядит на мобильных (если можно определить)
- Состояния интерфейса: loading, empty state, error — как обрабатываются
- Onboarding: как новый пользователь знакомится с продуктом
- Доступность (a11y): есть ли aria-атрибуты, контраст, keyboard navigation

### УРОВЕНЬ 4: Контентный и стилистический анализ
- Тон коммуникации (ToV): формальный/дружеский/экспертный
- Копирайтинг: заголовки, CTA-кнопки, описания — что работает, что нет
- Визуальный сторителлинг: как используются изображения, видео, иллюстрации
- Социальное доказательство: отзывы, кейсы, логотипы клиентов, цифры
- SEO-оптимизация: meta-теги, структура URL, заголовки H1-H3

### УРОВЕНЬ 5: Бизнес-модель
- Модель монетизации: SaaS подписка / freemium / one-time / marketplace
- Ценообразование: тарифы, позиционирование, якорная цена
- Воронка продаж: от первого касания до оплаты — сколько шагов
- Retention-механики: что удерживает пользователя (уведомления, привычки, данные)
- Виральность: есть ли механизмы рекомендаций, реферальная программа
- Unit-экономика (предположения): LTV, CAC, churn — что можно вывести

### УРОВЕНЬ 6: Целевая аудитория
- Основной сегмент: кто использует продукт (B2B/B2C, размер, отрасль)
- Персона пользователя: должность, задачи, боли, бюджет
- Jobs-to-be-Done: какую работу выполняет продукт для пользователя
- Альтернативы: что использует ЦА, если не этот продукт (включая "ничего"/"Excel")
- Барьеры входа: что мешает начать использовать

### УРОВЕНЬ 7: Конкурентное позиционирование
- В чем уникальность продукта (USP)
- Какие конкуренты существуют и чем они отличаются
- Матрица позиционирования: цена vs. функциональность
- Моат (защитный ров): что мешает скопировать продукт

### Вводные: анализ сервиса FreelanceHub (вымышленный конкурент TaskFlow)

**URL:** freelancehub.io (вымышленный)
**Описание:** Трекер задач и проектов для фрилансеров с таймером, инвойсами и отчетами для клиентов.

---

#### УРОВЕНЬ 1: Структурный анализ

**Резюме:** Классическая SaaS-структура с dashboard-центричной навигацией. 3 клика до создания первой задачи — хороший time-to-value.

Структура:
- Landing page → Sign up → Onboarding (3 шага) → Dashboard
- Dashboard: Проекты / Задачи / Таймер / Инвойсы / Отчеты
- Боковая навигация с группировкой по проектам
- Быстрые действия: "+" в header для создания задачи/проекта

User flows:
1. Новый пользователь: Landing → Sign up (email) → Выбор шаблона проекта → Dashboard
2. Ежедневный: Login → Dashboard → Выбор задачи → Старт таймера → Стоп → Автоотчет
3. Биллинг: Проект → Инвойс → Preview → Отправка клиенту по email

---

#### УРОВЕНЬ 2: Технический стек

| Компонент | Предположение | Признаки |
|-----------|--------------|----------|
| Frontend | React + Next.js | Структура URL (`/app/dashboard`), SSR-поведение, meta-теги `next` |
| CSS | Tailwind CSS | Утилитарные классы в DOM (`flex`, `gap-4`, `rounded-lg`) |
| UI-kit | Собственный, похож на shadcn/ui | Компоненты Button, Card, Dialog стандартной структуры |
| Backend | Node.js | API-эндпоинты через `/api/`, JSON-ответы |
| БД | PostgreSQL (Supabase) | Заголовки `x-supabase` в network, auth через `@supabase/auth-helpers` |
| Хостинг | Vercel | Заголовки `x-vercel-id`, домен на vercel.app до кастомного |
| Платежи | ЮKassa | Виджет оплаты с редиректом на `yookassa.ru`, характерные параметры `shopId` |
| Аналитика | Mixpanel + Hotjar | Скрипты в head |
| Auth | Supabase Auth (email + Google OAuth) | Форма входа с двумя вариантами |

---

#### УРОВЕНЬ 5: Бизнес-модель

**Резюме:** Freemium с ограничением на количество проектов. Конверсия в платный — через потолок бесплатного плана.

| Тариф | Цена | Ограничения |
|-------|------|-------------|
| Free | 0 | 2 проекта, 1 клиент, без инвойсов |
| Pro | $12/мес | Безлимитные проекты, инвойсы, отчеты |
| Team | $29/мес | + команда до 5, общие проекты, API |

Воронка: Бесплатная регистрация → использование 2-3 недели → достижение лимита → upgrade popup → оплата.

Retention: ежедневный таймер создает привычку + данные (часы, проекты) создают switching cost.

#### УРОВЕНЬ 8: Оценка и рекомендации

| Критерий | Оценка /10 | Комментарий |
|----------|-----------|-------------|
| Техническая реализация | 7 | Современный стек, но медленная загрузка dashboard (3+ сек) |
| UX/UI дизайн | 8 | Чистый интерфейс, но onboarding слишком длинный |
| Контент и копирайтинг | 6 | Слабые CTA, generic описания на лендинге |
| Бизнес-модель | 7 | Freemium работает, но Free-план слишком ограничен |
| Product-Market Fit | 7 | Решает реальную проблему, но конкурентов много |
| Масштабируемость | 6 | Нет API для интеграций, только ручной ввод |
| **Итого** | **6.8** | Добротный продукт без killer feature |

**ТОП-5 сильных сторон:**
1. Связка таймер + инвойс в одном продукте — закрывает полный цикл фрилансера
2. Чистый минималистичный UI без перегрузки
3. Быстрый onboarding — от регистрации до первой задачи за 2 минуты
4. Автоматическая генерация отчетов из залогированного времени
5. Email-нотификации с дайджестом продуктивности

**ТОП-5 слабых сторон:**
1. Нет мобильного приложения — фрилансеры работают с телефона
2. Инвойсы только в долларах — нет локализации валют (рубли, тенге)
3. Нет интеграции с банками и бухгалтерией
4. Отсутствует Kanban-доска — только список задач
5. Нет AI-функций (оценка времени, приоритизация, умные напоминания)

**3 возможности для конкурента (TaskFlow):**
1. AI-оценка времени на задачу на основе истории — экономия 30 минут в день на планировании
2. Интеграция с ЮKassa и российскими банками — инвойсы в рублях с автосверкой оплат
3. Telegram-бот для управления задачами и таймером — не нужно открывать веб-приложение

**Вердикт:** Продукт решает реальную проблему, но уязвим по локализации и мобильности. TaskFlow может занять нишу, сделав ставку на российский рынок (ЮKassa, рублевые инвойсы, Telegram-интеграция) и AI-функции (умная приоритизация, оценка времени).
