## КЛЮЧЕВОЙ КОНТЕКСТ

- **Дневная модель (КАНОН):** день закрыт = за ОДНУ локальную дату сданы ВСЕ 4 практики
  (фото креста + молитва + местописания-видео + пересказ-аудио); 7 закрытых дней → блок.
  Все 4 — ежедневные. Даты = `studentLocalToday` (пояс города, дефолт Бали). Эталон
  экранов — `api/m/cross-photo`. Подробности и частые баги — в `CLAUDE.md` (раздел
  «Дневные практики») и `memory/project_daily_day_model_canonical.md`.
- **ТАЙМЗОНЫ (канон, 2026-07-01):** любая per-student дата (сегодня/стрик/закрытие дня) —
  ТОЛЬКО через `studentLocalToday`/`localTodayStr` (`cities.timezone`), НЕ `baliToday()`/UTC.
  `computeActivity` требует `today` параметром. См. `memory/feedback_per_student_local_timezone.md`.
  Граница дня — строго полночь пояса города (сдвиг на 4:00 предлагали — Михаил ОТКЛОНИЛ).
- **Эталон фото креста:** Storage `block-resources/cross-reference/{order}.jpg`. Блок 1 (Малый
  Крест) = `1.jpg` (свой); блоки 2–10 = полный «Крест» (одинаковый `2..10.jpg`). ИИ качает по order.
- **Блок-гейт:** блок N открыт при ≥7 закрытых дней предыдущего блока (`is_block_unlocked`,
  чисто по прохождению) ИЛИ `can_skip_block_lock=true` (персонал). Проверено — багов нет.
- **Кто есть кто:** Михаил (sleezard@gmail.com) — super_admin, is_protected,
  единственный видит скрытых (`hidden_from_tracking`). Эля Ходус (@elyaforlife) — admin.
  Alex Magnier (@alex_magnier) — city_leader Москвы (city_id=10). Оля Мелешина
  (@Omeleshinka) — скрытый ученик (видит только владелец).
- **Замок владельца `owner_locked` (2026-07-02):** тест-аккаунты «Крест Капсула Тест»
  (@krest777237, город Москва с 01.07) и «Миша Моряк» (@crosscapsule, Бали) менять может
  ТОЛЬКО владелец — гард `lib/admin/locked.ts` во всех мутирующих путях панели и бота,
  fail-closed. Снять/поставить: `UPDATE profiles SET owner_locked=... WHERE id=...`.
- **Сессия панели сверяется с БД на каждый запрос** (guard.ts, 2026-07-02): роль/город
  берутся свежими; `getPanelSessionFromReq` теперь async — НЕ забывать await в новых роутах.
- **Аудит ролей — триггер БД** `zz_log_role_change` (changed_by=NULL значит «вне панели»);
  привилегированные колонки profiles защищены от RLS-клиентов триггером `a_protect_privileged_columns`.
- **Прод-бот:** `@cross_notify_bot`. Токен ротирован 2026-06-30 (старый был утечкой в
  `js/config.js`, убран). Вебхук: `/api/telegram/webhook`; поставить из браузера —
  `GET /api/telegram/set-webhook` (super_admin или `?key=CRON_SECRET`).
- **SQL на проде** (без CLI): Supabase Management API,
  `POST https://api.supabase.com/v1/projects/aejhlmoydnhgedgfndql/database/query`,
  токен `sbp_...` из `.mcp.json`, тело строй через `jq -Rn --rawfile q FILE '{query:$q}'`.
- **Правила:** только русский; no `alert/confirm` (toast/inline); TS strict без `any`;
  миграции только через `supabase/migrations/`; коммит/деплой как обычно (git push в main).
