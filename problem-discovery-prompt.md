# 0-03 | Формулировка ключевой проблемы продукта

> **Тип:** Инструкция (промпт для Claude AI)
> **Модуль:** 0 — Подготовка и анализ
> **Время применения:** 15-25 минут
> **Результат:** Четкая формулировка проблемы, корневой причины, последствий, решения и ценности — готовый фундамент для PROJECT_IDEA.md и спецификации

---

## Философия этого документа

Этот файл — **готовый промпт для Claude AI**. Вы не пишете его сами — копируете и используете.

**Принцип работы:**
- Вы — архитектор и коммуникатор
- Claude — исполнитель: читает промпт, делает работу
- Ваша задача — описать контекст и скопировать правильный промпт

**Workflow:**
```
Вы → копируете промпт → загружаете в Claude AI (web)
       ↓
       прикрепляете свой контекст (черновая идея / описание проблемы)
       ↓
Claude → выполняет задачу по промпту
       ↓
Вы → получаете результат → используете в проекте
```

Промпт уже отлажен. Не нужно его улучшать с первого раза — просто используйте.

---

## Что это и зачем

Большинство продуктов проваливаются не из-за плохого кода, а из-за того, что решают несуществующую проблему. Или решают реальную проблему, но не могут её внятно объяснить пользователю.

Этот промпт помогает пройти **Problem Discovery** — процесс формулировки проблемы по фреймворку:

```
Боль → Корневая причина → Последствия → Решение → Ценность
```

На выходе ты получаешь:
- Четко сформулированную проблему, которую решает продукт
- Понимание корневой причины (почему проблема существует)
- Количественные последствия (сколько стоит НЕ решать проблему)
- Формулировку решения через ценность для пользователя
- Готовый текст для раздела "Какую проблему решаем?" в PROJECT_IDEA.md
- Основу для value proposition и маркетинговых текстов

---

## Когда использовать

- **Перед написанием PROJECT_IDEA.md** — это должен быть самый первый шаг
- **Когда "размытая" идея** — есть ощущение, что что-то нужно рынку, но не можешь сформулировать
- **При получении заказа** — клиент говорит "хочу приложение", но не может объяснить зачем
- **При пивоте** — продукт не взлетает, нужно переосмыслить проблему
- **Перед питчем инвестору/клиенту** — нужна убедительная формулировка "зачем"
- **После анализа рынка** (`0-02`) — чтобы превратить данные о конкурентах в четкую проблему

Ты — продуктовый стратег, специализирующийся на problem discovery для цифровых продуктов. Помоги мне сформулировать ключевую проблему, которую решает мой продукт.

## ЗАДАЧА

Проведи полный Problem Discovery по 6 этапам. На каждом этапе задавай себе проверочные вопросы. Если данных недостаточно — формулируй гипотезы с пометкой "гипотеза, требует валидации".

### ЭТАП 1: Идентификация боли (Pain Identification)

Определи и сформулируй конкретные боли целевой аудитории:

**1.1. Составь список из 5-10 болей**, связанных с темой продукта. Для каждой боли:
- Опиши ситуацию, в которой боль проявляется (конкретный сценарий из жизни)
- Определи частоту (ежедневно / еженедельно / ежемесячно / при событии)
- Оцени интенсивность (1-10): насколько сильно это беспокоит
- Определи: осознает ли ЦА эту боль (осознанная / латентная)

**1.2. Ранжируй боли** по формуле:
```
Приоритет = Интенсивность × Частота × Готовность_платить
```

**1.3. Выбери ТОП-3 боли** — это ядро твоего продукта.

**Проверочные вопросы:**
- Если я подойду к представителю ЦА и опишу эту боль — он кивнет? Или скажет "ну, не то чтобы проблема"?
- Тратит ли ЦА время/деньги на решение этой боли прямо сейчас?
- Искал ли кто-то решение этой проблемы? (проверить через Google Trends, форумы, Telegram-чаты)

### ЭТАП 2: Корневой анализ (Root Cause Analysis)

Для каждой из ТОП-3 болей проведи анализ "5 Почему" (5 Whys):

**Формат:**
```
Боль: [описание]
  Почему? → [причина 1]
    Почему? → [причина 2]
      Почему? → [причина 3]
        Почему? → [причина 4]
          Почему? → [КОРНЕВАЯ ПРИЧИНА]
```

После анализа:
- Определи, какие корневые причины повторяются для разных болей
- Выяви системную проблему — единый корень нескольких симптомов
- Определи, можно ли технически устранить корневую причину (да / частично / нет)
- Если корневую причину нельзя устранить — определи, можно ли устранить последствия

**Классификация корневых причин:**
- **Информационный дефицит** — ЦА не знает чего-то важного → решение: дать информацию/аналитику
- **Процессный хаос** — нет системы, все вручную → решение: автоматизация
- **Инструментальный пробел** — нет подходящего инструмента → решение: создать инструмент
- **Экономический барьер** — существующие решения слишком дорогие → решение: доступная альтернатива
- **Когнитивная перегрузка** — слишком сложно → решение: упрощение

### ЭТАП 3: Последствия бездействия (Cost of Inaction)

Посчитай, сколько стоит НЕ решать проблему. Это ключевой аргумент для продажи продукта.

**3.1. Прямые потери:**
- Потери времени: сколько часов в неделю/месяц тратится на обходные решения
- Финансовые потери: упущенная выручка, штрафы, переплаты
- Потеря клиентов/заказов: из-за отсутствия системы

**3.2. Косвенные потери:**
- Стресс и выгорание: влияние на продуктивность и здоровье
- Упущенные возможности: что мог бы делать, если бы не тратил время на рутину
- Конкурентный проигрыш: как это влияет на позицию относительно конкурентов

**3.3. Кумулятивный эффект:**
- Посчитай потери за 1 месяц → 6 месяцев → 1 год
- Представь в рублях (или часах, если время — основная валюта ЦА)

**Формат итога:**
```
"Не решая эту проблему, [ЦА] теряет [X часов/рублей] в месяц,
что за год составляет [Y], при этом [качественные последствия]."
```

### ЭТАП 4: Формулировка решения (Solution Framing)

Сформулируй решение НЕ как набор функций, а как трансформацию:

**4.1. Формула трансформации:**
```
ИЗ: [текущее болезненное состояние — конкретно, с цифрами]
В:  [желаемое состояние — конкретно, с цифрами]
ЧЕРЕЗ: [механизм/подход — не функции, а принцип]
ЗА:  [временные рамки — когда ЦА увидит первый результат]
```

**4.2. Принцип решения:**
- Определи ключевой принцип (НЕ функции): автоматизация / централизация / AI-анализ / упрощение
- Объясни, почему этот принцип работает лучше существующих альтернатив
- Сформулируй в одном предложении: "Мы решаем [проблему] через [принцип], что дает [результат]"

**4.3. Минимальный proof of concept:**
- Какая ОДНА функция доказывает, что решение работает?
- Что должен увидеть пользователь в первые 5 минут использования, чтобы сказать "вау, это то, что мне нужно"?

**Проверочные вопросы:**
- Если убрать AI/автоматизацию — остается ли ценность? (если нет — продукт хрупкий)
- Можно ли объяснить решение бабушке за 30 секунд?
- Существует ли "момент магии" — конкретная точка, где пользователь видит ценность?

### ЭТАП 5: Ценностное предложение (Value Proposition)

На основе предыдущих этапов, сформулируй ценность продукта в нескольких форматах:

**5.1. Elevator Pitch (30 секунд):**
```
[Продукт] помогает [кому] [делать что],
чтобы [получить какой результат],
вместо [текущей альтернативы, которая не работает].
```

**5.2. Развернутое ценностное предложение (2-3 предложения):**
Включает проблему, решение, результат и отличие от конкурентов.

**5.3. Заголовок для лендинга:**
2-3 варианта заголовков, которые можно использовать на главной странице.

**5.4. Антипод — "мы НЕ":**
Четко скажи, чем продукт НЕ является. Это помогает отстроиться и избежать завышенных ожиданий.

### ЭТАП 6: Валидационный чеклист

Проверь сформулированную проблему по критериям:

| Критерий | Да/Нет | Комментарий |
|----------|--------|-------------|
| **Реальность:** Проблема существует у реальных людей, а не только в моей голове? | | |
| **Частота:** Проблема возникает регулярно (не раз в год)? | | |
| **Интенсивность:** Люди готовы платить за решение (не просто "было бы неплохо")? | | |
| **Доступность:** Я могу найти и привлечь этих людей? | | |
| **Решаемость:** Проблему можно решить техническими средствами? | | |
| **Масштаб:** Людей с этой проблемой достаточно для бизнеса (1000+ платящих)? | | |
| **Timing:** Почему СЕЙЧАС — не 5 лет назад и не через 5 лет? | | |
| **Уникальность:** Мое решение чем-то лучше существующих? | | |
| **Объяснимость:** Могу объяснить проблему и решение за 30 секунд? | | |
| **Измеримость:** Могу доказать ценность цифрами (время, деньги)? | | |

Если 7+ "Да" — проблема сформулирована хорошо. Если менее 5 — нужно переосмыслить.

## Пример результата

> Ниже — сокращенный пример Problem Discovery для вымышленного продукта TaskFlow.

### Вводные

**Продукт:** TaskFlow — трекер задач для фрилансеров.
**ЦА:** Русскоязычные фрилансеры (дизайнеры, разработчики, копирайтеры).
**Функции:** Трекер задач, таймер, инвойсы в рублях, AI-оценка времени, Telegram-бот.

---

#### ЭТАП 1: Идентификация боли

**ТОП-3 боли русскоязычного фрилансера:**

| # | Боль | Ситуация | Частота | Интенсивность | Осознанность |
|---|------|----------|---------|--------------|-------------|
| 1 | Не знаю, сколько времени займет задача | Клиент спрашивает "когда будет готово?", я называю случайную цифру, потом не успеваю | Каждый проект | 9/10 | Осознанная |
| 2 | Трачу время на рутину вместо работы | Каждый понедельник — 2 часа на составление счетов, отчетов, переписку с клиентами | Еженедельно | 7/10 | Частично |
| 3 | Не понимаю, сколько реально зарабатываю | Деньги приходят хаотично, не могу посчитать часовую ставку, не знаю какие проекты прибыльные | Ежемесячно | 8/10 | Латентная |

---

#### ЭТАП 2: Корневой анализ

**Боль #1: Не знаю, сколько времени займет задача**

```
Почему не знаю? → Нет данных о прошлых проектах
  Почему нет данных? → Не трекаю время на задачах
    Почему не трекаю? → Существующие трекеры — отдельные приложения, лень переключаться
      Почему лень? → Нет инструмента, встроенного в мой рабочий процесс (Telegram)
        КОРНЕВАЯ ПРИЧИНА: Инструментальный пробел — нет трекера,
        который живет там, где живет фрилансер (Telegram), а не в отдельном веб-приложении
```

**Боль #2: Трачу время на рутину**

```
Почему трачу? → Вручную составляю счета, отчеты, акты
  Почему вручную? → Нет единой системы, где все данные
    Почему нет системы? → Использую 5 разных инструментов (таблицы, мессенджер, банк, почту, заметки)
      Почему 5 инструментов? → Ни один не объединяет все нужное фрилансеру
        КОРНЕВАЯ ПРИЧИНА: Процессный хаос — данные разбросаны по 5 инструментам,
        нет единого места "задачи + время + деньги"
```

**Системная проблема:** У русскоязычного фрилансера нет единого инструмента, который живет в его экосистеме (Telegram + рублевые платежи) и объединяет управление задачами, учет времени и биллинг.

---

#### ЭТАП 3: Последствия бездействия

**Прямые потери:**
- Время: 8-12 часов/мес на ручной учет, отчеты, счета
- Деньги: занижение стоимости проектов на 20-30% из-за неточной оценки времени
- Клиенты: 1-2 потерянных проекта в год из-за сорванных дедлайнов (неверная оценка)

**Количественно:**

| Потеря | За месяц | За год |
|--------|---------|--------|
| Время на рутину | 10 часов | 120 часов |
| Недополученная оплата (заниженные оценки) | 15,000 руб | 180,000 руб |
| Потерянные клиенты (сорванные дедлайны) | — | 50,000-100,000 руб |
| **Итого** | **~25,000 руб** | **~300,000 руб** |

> "Не решая эту проблему, фрилансер теряет 10 часов и 25,000 рублей ежемесячно на ручной рутине и неточных оценках, что за год составляет 300,000 рублей — при этом испытывает хронический стресс от хаоса в задачах и финансах."

---

#### ЭТАП 4: Формулировка решения

**Формула трансформации:**

```
ИЗ: 5 разрозненных инструментов, 10 часов рутины в месяц,
    случайные оценки сроков, хаос в финансах

В:  Один инструмент в Telegram, 1 час рутины в месяц,
    AI-оценки на основе реальных данных, прозрачная финансовая картина

ЧЕРЕЗ: Централизация (задачи + время + деньги в одном месте)
       + AI-анализ (предсказание сроков на основе истории)
       + автоматизация (инвойсы генерируются из залогированного времени)

ЗА: Первый результат через 5 минут (запуск таймера через Telegram-бота),
    полная картина — через 2 недели использования
```

**Момент магии:** Пользователь запускает таймер на задачу в Telegram-боте (`/start Дизайн лендинга для ООО Продуктив`). Через 3 часа останавливает. TaskFlow автоматически предлагает: "Готов инвойс на 4,500 руб (3 ч × 1,500 руб/ч). Отправить клиенту?" Один клик — счет уходит на email клиента с кнопкой оплаты через ЮKassa.

#### ЭТАП 5: Ценностное предложение

**Elevator Pitch:**

> TaskFlow помогает фрилансерам управлять задачами, временем и деньгами из одного Telegram-бота, чтобы тратить на рутину 1 час в месяц вместо 10, вместо жонглирования пятью разными инструментами и Excel-таблицами.

**Заголовки для лендинга:**

1. "Управляй фрилансом из Telegram. Задачи, время, деньги — в одном боте."
2. "Фрилансер, верни себе 10 часов в месяц. TaskFlow автоматизирует рутину."
3. "Хватит считать в Excel. TaskFlow знает, сколько ты зарабатываешь на каждом проекте."

**Мы НЕ:**
- Мы не Jira и не ClickUp — у нас нет спринтов, бэклогов и Gantt-диаграмм
- Мы не бухгалтерская программа — мы не заменяем 1С или банковское приложение
- Мы не для команд — мы для одного фрилансера или пары партнеров

#### ЭТАП 6: Валидационный чеклист

| Критерий | Да/Нет | Комментарий |

| Частота | Да | Ежедневно (задачи/таймер), еженедельно (отчеты), ежемесячно (инвойсы) |
| Интенсивность | Да | Готовы платить 500-1500 руб/мес за экономию 10 часов |

| Уникальность | Да | Никто не делает трекер в Telegram с рублевыми инвойсами |

| Измеримость | Да | 10 часов/мес экономии, 300,000 руб/год неупущенной выручки |

# Problem Statement: TaskFlow

## Проблема
Русскоязычные фрилансеры тратят 10+ часов в месяц на ручной учет задач,
времени и финансов, используя 5 разрозненных инструментов, при этом
систематически занижают стоимость своей работы из-за отсутствия данных.

## Кто страдает
Самозанятые и ИП в России (9+ млн человек): дизайнеры, разработчики,
копирайтеры, маркетологи — те, кто работает на себя и ведет 3-10 проектов
одновременно.

## Корневая причина
Инструментальный пробел: нет единого решения, которое живет в экосистеме
фрилансера (Telegram + рублевые платежи) и объединяет задачи, учет времени
и биллинг. Существующие решения либо западные (без рублей и ЮKassa),
либо корпоративные (избыточные для одного человека).

## Цена бездействия
~25,000 руб/мес (300,000 руб/год): 10 часов рутины + недополученная
оплата из-за неточных оценок + потерянные клиенты из-за сорванных дедлайнов.

## Решение
Централизация управления фрилансом в одном инструменте с Telegram-ботом
как основным интерфейсом, AI-оценкой времени на основе истории, и
автоматической генерацией инвойсов в рублях с оплатой через ЮKassa.

## Ценность
TaskFlow экономит фрилансеру 10 часов в месяц и увеличивает доход
на 20-30% за счет точных оценок времени и прозрачной финансовой картины.

## Доказательство
MVP: Telegram-бот с таймером → автоматический инвойс → оплата через ЮKassa.
Если пользователь за 5 минут запустил таймер и через час получил готовый
счет — ценность доказана.

## Риски и неизвестные
- Готовность фрилансеров платить 500-1500 руб/мес (требует валидации через опрос)
- Точность AI-оценки времени на малом объеме данных (первые 2-4 недели)
- Интеграция с ЮKassa для самозанятых (техническая проверка API)

## Последовательность работы с документами модуля 0

```
0-01 Обратный инжиниринг    →  Понимаешь, как устроены конкуренты
         ↓
0-02 Анализ рынка            →  Видишь полную картину: кто, сколько, где пробелы
         ↓
0-03 Формулировка проблемы   →  Четко знаешь, какую боль решаешь и почему
         ↓
     PROJECT_IDEA.md          →  Готов написать идею проекта для спецификации
         ↓
     SPEC.md                  →  Полная спецификация для Claude Code
```

Каждый следующий документ опирается на предыдущий. Результаты `0-01` и `0-02` можно прикрепить к чату при работе с `0-03` — Claude учтет данные о конкурентах и рынке при формулировке проблемы.
