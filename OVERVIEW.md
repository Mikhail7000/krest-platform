# КРЕСТ — обзор платформы

Внутренняя платформа церкви для управляемого ученичества по курсу «Крест» (10 блоков).
Одно приложение обслуживает три аудитории: учеников, кураторов, руководство.

---

## Что это и как работает (коротко)

Ученик заходит через **Telegram (мини-приложение)** или браузер, проходит курс из 10 блоков.
Каждый блок устроен по **дневной модели**: чтобы «закрыть день», нужно выполнить набор практик; набрав достаточно закрытых дней — открывается следующий блок. Куратор сопровождает свою группу, руководство видит всю платформу.

**Аудитории:**
- **Ученик** — учится (Telegram + браузер).
- **Куратор** — ведёт свою группу.
- **Админ / Супер-админ** — управляют платформой.

---

## Техническая начинка (стек)

- **Фронтенд + мини-приложение + админка:** Next.js 16 (App Router), React 19, TypeScript, Tailwind, shadcn/ui. Один код на все три интерфейса.
- **Бэкенд:** Supabase (PostgreSQL 15, Auth, хранилище файлов, Row-Level Security) + серверные API-маршруты Next.js.
- **Видео:** Kinescope (встроенный плеер с защитой от перемотки — нужно посмотреть ≥95%).
- **ИИ:** Anthropic Claude (`claude-sonnet-4-6`) — тренажёр стихов/местописаний. Транскрипция голосовых от учеников — **Deepgram Nova-2** (распознаёт русскую речь, голосовые → текст).
- **Уведомления:** Telegram Bot API + email (Resend).
- **Хостинг:** Vercel (приложение) + Supabase Cloud (база). Деплой автоматический при пуше кода.

## Курс и логика прохождения

- **10 блоков** (Малый Крест → … → 5 Уверенностей) + вводный модуль.
- **Дневная модель блока.** День считается «закрытым», когда ученик за одни сутки выполнил все обязательные практики:
  1. **Писать крест** (фото);
  2. **Молитва** (отметка);
  3. **Пересказ** (аудио);
  4. **Местописания** (видео-кружок / запись).
  На блок также: **квиз** и **практика пятницы**.
- **Открытие следующего блока:** когда в текущем блоке закрыто **≥ 7 дней**.
- **Видео** в блоке смотрятся без перемотки (Kinescope, ≥95%).
- **Тренажёр** (ИИ): помогает заучивать стихи/местописания перед сдачей.
- **Лента** (community): ученики делятся переживаниями — текст, голос, видео-кружок, фото.
- **Трекинг:** общий рейтинг-лидерборд по очкам и закрытым дням (поиск по имени/нику).
- **Профиль:** аватар, локация, оформление (темы: светлая / тёмная / звёзды / **розовая** — авто для девушек).

**Экзамены:**
- 10 сдач блоков куратору (офлайн);
- 1 промежуточный экзамен после блока 5 (у другого куратора);
- 1 финальный экзамен после блока 10 (у админа) → достижение «Мастер Креста».

---

## Как работает ИИ

- **Тренажёр стихов/местописаний:** Claude проверяет пересказ ученика, даёт обратную связь.
- **Голосовые сообщения** учеников распознаются через Deepgram (русская речь → текст) — чтобы куратор/система могли работать с содержанием.
- AI-ассистент-наставник для блоков — в планах (post-MVP).

---

## Как работает чат-бот (Telegram)

Бот — это и **точка входа** в мини-приложение, и **канал уведомлений**, и **пульт управления** для руководства.

- **Вход:** ученик открывает мини-приложение прямо из бота.
- **Доступ новых:** незнакомый пользователь оставляет заявку → она уходит **всем администраторам** в бот с кнопками: «Впустить учеником» / «Сделать куратором» / «Отклонить». Заявки также видны в веб-панели.
- **Команды для руководства** (в боте): вход в веб-дашборд, список учеников, статистика, карточка ученика по нику, список кураторов, массовая привязка учеников к куратору.
- Куратору команды показывают **только его** учеников.

---

## Как работает веб-панель (`/panel`)

Админка в браузере. **Вход** — по ссылке из Telegram-бота (одноразовая, 10 минут) → подписанная cookie-сессия на 7 дней. Логин/пароль не нужен.

Разделы: **Обзор** (статистика), **Заявки** (на доступ), **Ученики**, **Кураторы**, **Города**.

- **Обзор:** сводка по платформе — ученики, кураторы, города, прогресс по блокам, топ по непрерывным дням, «застрявшие».
- **Заявки:** одобрить/отклонить новых (дубль кнопок бота), история решений (кем и когда).
- **Ученики:** список с поиском и сортировкой по любому столбцу; карточка ученика с прогрессом по блокам; смена роли/куратора; добавление по нику.
- **Кураторы:** кураторы и админы, их группы; смена ролей (куратор↔ученик↔админ — управление админами только у супер-админа).
- **Города:** управление гео.

## Доступы и функции по ролям

| Возможность | Ученик | Куратор | Админ | Супер-админ |
|---|:--:|:--:|:--:|:--:|
| Проходить курс, лента, трекинг, тренажёр, профиль | ✅ | — | — | — |
| Веб-панель | — | ✅ (ограниченная) | ✅ | ✅ |
| Видеть учеников | свою группу | **только своих** | всех | всех |
| Принимать сдачи / вести группу | — | ✅ | ✅ | ✅ |
| Заявки на доступ (решать) | — | — | ✅ | ✅ |
| Назначать кураторов, прикреплять учеников | — | — | ✅ | ✅ |
| Менять роли | — | — | ✅ (кроме админ-уровня) | ✅ |
| CRUD городов, назначать админов, передача прав | — | — | — | ✅ |

- **Куратор в веб-панели** видит **только «Обзор» и «Ученики» своей группы** — без заявок, кураторов, городов и админ-действий.
- **Супер-админы:** Михаил, Алекс, Эля, Игорь. Владелец (Михаил) защищён флагом — его роль нельзя изменить через интерфейс.
- Все изменения ролей пишутся в журнал аудита.

## Уведомления в боте: время, часовые пояса, напоминания

**Ежедневные напоминания ученику** (если он сегодня не заходил), время **Бали (UTC+8)**:

| Время (Бали) | Тон |
|---|---|
| **18:00** | мягкое: «не забудь зайти» |
| **20:00** | личное: «я и сам учился по вечерам…» |
| **21:00** | если так и не зашёл — последнее за день |

- Напоминание уходит **только тем, кто не открывал приложение в этот день**; на каждом этапе — не чаще одного раза.
- Серия активности («стрик») обнуляется для тех, кто неактивен >7 дней.

**Уведомления руководству/куратору:**
- Новая заявка на доступ → всем админам.
- Новый сабмишен ученика, готовность к сдаче, пройденные экзамены → по матрице уведомлений.
- Простой ученика >3 дней → куратору (в ближайшем обновлении).

> **Важно про пояса:** сейчас «день» и напоминания считаются по времени **Бали/UTC**. Переход на **локальный пояс каждого ученика** (день закрывается и следующий открывается в 00:00 по его поясу, с уведомлением) — в ближайшем плане.

## Что сделано и что в планах (кратко)

**Готово:** курс 10 блоков, дневная модель, видео без перемотки, лента, трекинг с поиском, тренажёр, темы оформления (вкл. розовую), заявки на доступ (бот + веб-панель с историей), управление ролями с аудитом, веб-панель руководства, ограниченная веб-панель куратора, ежедневные напоминания (18/20/21 Бали).

**В планах:** уведомления куратору о событиях его учеников + настройки частоты; фокус на отдельных учениках; отчёты по выбранным ученикам; локальные часовые пояса учеников; богатый профиль (о себе, как пришёл к Богу, медиа); ускорение приложения.
