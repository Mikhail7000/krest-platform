# КРЕСТ — обзор платформы

Внутренняя платформа церкви для **управляемого ученичества** по курсу «Крест» (10 блоков).
Одно приложение (один код) обслуживает три аудитории: учеников, кураторов, руководство.
Старт — Бали; платформа поддерживает 8 стран и 19+ городов (открываются по мере назначения кураторов).

## 1. Аудитории и роли

| Роль | Кто | Что делает |
|---|---|---|
| **Ученик** (`student`) | проходящий курс | Учится через Telegram-приложение или браузер |
| **Куратор** (`curator`) | наставник группы | Ведёт свою группу учеников, принимает сдачи |
| **Админ** (`admin`) | управляющий зоной | Назначает кураторов, прикрепляет учеников, решает заявки |
| **Супер-админ** (`super_admin`) | руководство | Полный доступ: роли, города, передача прав |

Супер-админы: Михаил, Алекс, Эля, Игорь. Владелец платформы (Михаил) помечен флагом `is_protected` — его роль нельзя изменить через интерфейс (только прямым доступом к базе).

## 2. Технологический стек

**Фронтенд / приложение / админка** (один код на всё):
- **Next.js 16** (App Router) + **React 19** + **TypeScript** (strict).
- **Tailwind CSS v4** + **shadcn/ui** + Framer Motion (анимации).
- Server Components по умолчанию (быстрый рендер на сервере).

**Бэкенд / данные:**
- **Supabase**: PostgreSQL 15, Auth, файловое хранилище (Storage), Row-Level Security (RLS), Realtime.
- Серверные **API-маршруты Next.js** (бизнес-логика, интеграции).

**Медиа и контент:**
- **Kinescope** — видео-плеер со встроенной защитой от перемотки.

**ИИ и распознавание:**
- **Anthropic Claude** (`claude-sonnet-4-6`) — тренажёр стихов/местописаний.
- **Deepgram Nova-2** — распознавание русской речи (голосовые ученика → текст).

**Уведомления:** Telegram Bot API + email (Resend).

**Хостинг:** Vercel (приложение) + Supabase Cloud (база). Деплой — автоматический при отправке кода в репозиторий.

## 3. Архитектура: три интерфейса

1. **Telegram мини-приложение** (`/m/*`) — основной интерфейс ученика. Открывается прямо из бота, работает и в браузере (паритет функций).
2. **Веб-панель** (`/panel`) — рабочее место кураторов и руководства.
3. **Telegram-бот** — точка входа, канал уведомлений и пульт управления (команды).

Все три используют общую базу и серверную логику. Чувствительные операции выполняются только на сервере (сервисный ключ базы в браузер не попадает).

---

## 4. Онбординг (вход нового ученика)

1. Пользователь открывает бота и запускает мини-приложение.
2. **Доступ:** если пользователь не в списке доступа — создаётся **заявка**. Она уходит **всем администраторам** в Telegram с кнопками «Впустить учеником / Сделать куратором / Отклонить» и одновременно появляется в веб-панели (раздел «Заявки»).
3. После одобрения ученик проходит онбординг: **язык → страна → город → куратор → личные данные**.
4. Города со статусом «Скоро» ставят ученика в лист ожидания до активации (назначения куратора).

## 5. Структура курса

**10 блоков** (последовательно):

| № | Блок | № | Блок |
|---|---|---|---|
| 1 | Малый Крест | 6 | Усилие Человека |
| 2 | Принцип Сотворения | 7 | Обетования и Исполнение |
| 3 | Коренная Проблема | 8 | Иисус Христос |
| 4 | Состояние Мира | 9 | Благословения Верующего |
| 5 | Состояние Неверующего | 10 | 5 Уверенностей |

Каждый блок включает: вводный экран, **основное видео** (Kinescope, без перемотки), доп. видео, форум-рефлексию (3 вопроса), конспект, ежедневные практики (см. ниже), местописания, прослушивание молитв (блок 1), сдачу куратору и практику «эпоха пятницы».

---

## 6. Дневная модель и открытие блоков

Прохождение блока построено по **дням**. Чтобы «**закрыть день**», ученик за одни сутки выполняет 4 ежедневные практики:

1. **Писать крест** — фото;
2. **Молитва** — отметка;
3. **Пересказ** — голосовая запись (аудио);
4. **Местописания** — видео-кружок / запись.

Плюс на блок: **квиз** (тест на знание) и практика **пятницы**.

- **Открытие следующего блока:** когда в текущем закрыто **≥ 7 дней**.
- **Тренажёр** (ИИ) помогает заучивать местописания/стихи перед сдачей.
- Для тестировщиков есть режим, снимающий ожидание дней (ускоренное прохождение).

> Технически «день» сейчас отсчитывается по времени **Бали (UTC+8)**. Переход на локальный часовой пояс каждого ученика (день закрывается в 00:00 по его поясу, с уведомлением) — в ближайшем плане.

## 7. Экзамены и достижения

**Иерархия экзаменов:**
- **10 сдач блоков** куратору — пункт «Сдача» в каждом блоке (офлайн, куратор подтверждает вручную).
- **1 промежуточный экзамен** после блока 5 (принимает другой куратор).
- **1 финальный экзамен** после блока 10 (принимает админ) → достижение **«Мастер Креста»** и открытие следующего курса.

**Достижения** оформлены в библейском стиле (по 1 Иоанна 2:12-14: «победил лукавого» и т. п.), отображаются в трекинге и профиле.

---

## 8. Функции ученика

- **Курс:** прохождение 10 блоков по дневной модели, видео без перемотки.
- **Тренажёр** стихов/местописаний (с ИИ-проверкой).
- **Лента** (community): публикации-переживания — **текст / голос / видео-кружок / фото**. Фото отображаются целиком в любом формате (вертикальные тоже) и открываются на весь экран по тапу. Можно удалить свою публикацию.
- **Трекинг:** общий рейтинг-лидерборд по очкам и закрытым дням; поиск по имени или нику; призовые места (золото/серебро/бронза).
- **Профиль:** аватар, локация, **оформление** — выбор темы: светлая / тёмная / звёзды / **розовая** (розовая применяется автоматически для девушек).
- **Напоминания** в бот, если сегодня не заходил (см. раздел 13).

## 9. Искусственный интеллект

- **Тренажёр стихов/местописаний:** Claude проверяет пересказ ученика и даёт обратную связь — помогает выучить перед сдачей.
- **Распознавание голосовых:** голосовые сообщения ученика автоматически переводятся в текст (Deepgram Nova-2, русская речь) — чтобы система и куратор работали с содержанием, а не только со звуком.
- **AI-наставник** для сопровождения по блокам — в планах (после MVP).

## 10. Чат-бот (Telegram)

Бот — это **вход** в приложение, **канал уведомлений** и **пульт управления** для руководства.

- **Вход ученика:** открытие мини-приложения из бота.
- **Заявки на доступ:** новый пользователь → заявка всем админам с inline-кнопками решения.
- **Команды руководства** (в боте):
  - вход в веб-дашборд (ссылка-автологин на 10 минут);
  - список учеников и их прогресс;
  - статистика платформы;
  - карточка ученика по нику;
  - список кураторов и их групп;
  - массовая привязка учеников к куратору по никам.
- Для **куратора** команды показывают **только его** учеников.

## 11. Веб-панель (`/panel`)

Браузерная админка. **Вход** — по одноразовой ссылке из Telegram-бота (живёт 10 минут) → подписанная cookie-сессия на 7 дней. Логин/пароль не нужны.

Разделы:

- **Обзор** — сводная статистика: число учеников/кураторов/админов, распределение по текущему блоку, по городам и странам, топ по непрерывным дням, список «застрявших» (нет активности >3 дней). Кнопка генерации отчёта.
- **Заявки** — новые запросы на доступ: одобрить учеником/куратором или отклонить (те же кнопки, что в боте), плюс **история решений** (кем и когда принято).
- **Ученики** — список с **поиском** и **сортировкой по любому столбцу** (имя, город, куратор, сдано, текущий блок, дней закрыто, дата). Карточка ученика: прогресс по каждому блоку (дней X/7, квиз, сдан ли). Действия: прямой выбор куратора в строке, смена роли/куратора, добавление по нику.
- **Кураторы** — кураторы и админы платформы, их города и группы; смена ролей (куратор↔ученик↔админ; управление админами — только у супер-админа). Массовая привязка учеников.
- **Города** — управление географией (статусы «активен / скоро»).

**Панель куратора** (ограниченная): куратор входит в ту же панель, но видит **только «Обзор» и «Ученики» своей группы** — без разделов «Заявки», «Кураторы», «Города» и без админ-действий. Видит исключительно своих учеников.

## 12. Доступы и функции по ролям

| Возможность | Ученик | Куратор | Админ | Супер-админ |
|---|:--:|:--:|:--:|:--:|
| Курс, лента, трекинг, тренажёр, профиль | ✅ | — | — | — |
| Вход в веб-панель | — | ✅ (урезанная) | ✅ | ✅ |
| Видит учеников | свою группу | **только своих** | всех | всех |
| Принимает сдачи, ведёт группу | — | ✅ | ✅ | ✅ |
| Решает заявки на доступ | — | — | ✅ | ✅ |
| Назначает кураторов, прикрепляет учеников | — | — | ✅ | ✅ |
| Меняет роли | — | — | ✅ (кроме админ-уровня) | ✅ |
| CRUD городов, назначение админов, передача прав | — | — | — | ✅ |

- **Куратор** в панели и в боте видит **исключительно свою группу** (никогда — чужих учеников).
- **Видимость как маркер прогрессии:** ученик видит свою группу; после сдачи всего курса — открывается более широкая видимость. Куратор видит свою группу + кураторов своего города.
- Все изменения ролей пишутся в **журнал аудита** (`role_change_log`).

## 13. Уведомления: время и часовые пояса

**Ежедневные напоминания ученику** (только если он сегодня **не открывал** приложение), время **Бали (UTC+8)**:

| Время (Бали) | Характер сообщения |
|---|---|
| **18:00** | мягкое: «не забудь зайти сегодня» |
| **20:00** | личное, тёплое: «я и сам учился по вечерам…» |
| **21:00** | последнее за день: «ещё не поздно» |

- На каждом этапе — не чаще **одного раза**; тем, кто уже заходил, напоминания не идут.
- Серия активности («стрик») обнуляется при простое >7 дней.
- Все отправленные уведомления логируются.

**Уведомления руководству / куратору:**
- Новая заявка на доступ → **всем администраторам** (в бот + в веб-панель).
- Действия в панели (смена роли и т. п.) → информируют админов.
- *(В ближайшем плане)* куратору: новый сабмишен его ученика, готовность к сдаче, простой ученика >3 дней, пройденные экзамены; плюс персональные настройки частоты этих уведомлений.

> **Часовые пояса:** сейчас «день» и напоминания считаются по **Бали (UTC+8)**. Переход на **локальный пояс каждого ученика** — в ближайшем плане (день закрывается и следующий открывается в 00:00 по его поясу, с уведомлением).

## 14. Безопасность и данные

- **Row-Level Security** на таблицах базы; ученик видит только свои записи, доступ разграничен ролью.
- **Сервисный ключ** базы — только на сервере, никогда в браузере.
- **Валидация Telegram:** подпись `initData` проверяется (HMAC) на каждом запросе мини-приложения.
- **Защита владельца:** флаг `is_protected` не даёт разжаловать владельца через интерфейс.
- **Аудит ролей:** каждое изменение роли фиксируется в журнале (кто, кого, когда, старая→новая роль).
- **Scoping куратора:** так как панель работает на сервере с полным доступом к базе, ограничение «куратор видит только своих» обеспечивается в коде на каждом маршруте данных.
- **XSS:** ввод ученика экранируется (React); контент руководства проходит очистку перед вставкой.

## 15. Инфраструктура и деплой

- **Vercel** — хостинг приложения; **Supabase Cloud** — база и хранилище.
- **Деплой автоматический:** отправка кода в основную ветку репозитория запускает сборку и публикацию.
- Изменения схемы базы — только через версионированные миграции.
- Запланированные задачи (cron): ежедневные напоминания (18:00 / 20:00 / 21:00 Бали), обнуление неактивных серий.
- Старт — **только Бали**; остальные города открываются по мере назначения кураторов.

## 16. Готово / в планах

**Готово:** курс из 10 блоков, дневная модель, видео без перемотки, тренажёр с ИИ, распознавание голосовых, лента (текст/голос/кружок/фото с полноэкранным просмотром), трекинг с поиском, темы оформления (включая розовую авто-для-девушек), заявки на доступ (бот + веб-панель с историей решений), управление ролями с аудитом, веб-панель руководства, **ограниченная веб-панель куратора** (видит только своих), ежедневные напоминания (18/20/21 Бали).

**В ближайших планах:** уведомления куратору о событиях его учеников + персональные настройки частоты; уведомление о простое ученика >3 дней; «фокус» куратора на отдельных учениках; отчёты по выбранным ученикам (куратор — по своим, админ — по всем); локальные часовые пояса учеников (день/напоминания по поясу ученика); богатый профиль (о себе, чем занимается, «как пришёл к Богу», медиа и ссылки); публичный профиль ученика из трекинга; ускорение приложения; AI-наставник по блокам.
