# Notes — личные заметки сессий

Здесь Михаил сохраняет важные ответы Claude, конспекты, методологии — всё, что хочется иметь под рукой и редактировать вручную.

## Как сохранить ответ

Сразу после того, как Claude дал важный ответ, скажи:

```
/save-last <имя-файла>
```

Например: `/save-last methodology-explained` → сохранит мой последний ответ в `notes/methodology-explained.md` с датой и заголовком.

## Как прочитать заметки

Любая фраза:
- «покажи все заметки» / «что у меня в notes»
- «прочитай notes/methodology-explained.md»
- «найди в notes слово subagents»

## Как редактировать / удалять

- «удали файл notes/X.md»
- «удали из notes/X.md абзац про Y»
- «оставь в notes/X.md только раздел Z, остальное сотри»
- «переименуй notes/X.md в notes/Y.md»

Claude сделает Edit/Write/Bash — ничего особого настраивать не надо.

## Структура (рекомендация)

Свободная — как удобно. Примеры имён:
- `methodology-explained.md` — объяснения концепций
- `decisions-2026-04-30.md` — принятые решения за дату
- `alex-lesson-XX-summary.md` — конспект урока
- `ideas-backlog.md` — идеи на потом

## Git

Папка коммитится в репозиторий по умолчанию. Если хочешь оставить заметки **только локально** (не пушить в GitHub) — добавь в корневой `.gitignore`:

```
notes/
```
