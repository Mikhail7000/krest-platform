# Скилл: Реализовать фичу

## Перед началом — уточняющие вопросы

Если спецификация неполная — задай вопросы ДО начала работы:
- Какие страницы затронуты (student/*.html, admin/*.html)?
- Нужны ли изменения в БД (новая колонка, таблица)?
- Какой субагент реализует (Frontend Developer / Database Architect)?

Без ответов не начинай.

## Порядок реализации

```
1. Анализ → прочитать CLAUDE.md + затронутые файлы
2. БД → делегировать Database Architect если нужна миграция
3. JavaScript → Frontend Developer реализует логику
4. UI → Frontend Developer создаёт/правит HTML/CSS
5. i18n → добавить переводы в js/config.js (RU + EN)
6. QA → запустить run-qa-review после реализации
```

## Что проверить перед отправкой

- [ ] requireAuth() / requireAdmin() на странице не убран
- [ ] Новые строки добавлены в T.ru и T.en в config.js
- [ ] Конспект не показан до форума (если затронут lesson.html)
- [ ] Используется toast() а не alert()
- [ ] Миграция с IF NOT EXISTS если затронута БД
- [ ] QA Review пройден
