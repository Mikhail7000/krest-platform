# Скилл: Generate FEATURE_SPEC.md

## Триггер
Пользователь говорит "напиши feature-spec для X" / "сделай fspec" / `/feature-spec`.

## Когда использовать

- Добавляется новый модуль в КРЕСТ (не покрытый SPEC.md)
- Расширение MVP после первого релиза
- Клиент попросил фичу, которой нет в исходной спецификации

**НЕ применять** если:
- Строим проект с нуля (используй полную SPEC.md)
- Изменения затрагивают >50% кодовой базы (это рефакторинг)

**Правило:** одна FEATURE_SPEC = одна фича. Не смешивать в одном документе.

## Что генерировать

Создать `docs/feature-specs/YYYY-MM-DD-{feature-name}.md` с 7 разделами:

### Шаблон

```markdown
# Feature: [Название]

> Проект: КРЕСТ
> Дата: YYYY-MM-DD
> Приоритет: Critical / High / Medium / Low
> Оценка: 1-2 часа / полдня / 1 день / 2-3 дня

---
```

## После генерации

3. Спросить: "Готов к реализации? Делегировать субагентам?"
4. Если да → запустить `implement-feature` skill
