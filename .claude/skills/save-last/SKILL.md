# Скилл: save-last

## Триггер
- `/save-last <имя-файла>` — пользователь хочет сохранить мой последний (предыдущий) ответ
- Без аргумента — спросить у пользователя желаемое имя файла

## Что делать

1. **Определить имя файла:**
   - Из аргумента после `/save-last` — берём первое слово как basename
   - Если аргумента нет — спросить «Под каким именем сохранить?» и подождать
   - Очистить от пробелов и спецсимволов: только `[a-z0-9-_]`, в нижнем регистре
   - Расширение `.md` добавить если его нет
   - Целевой путь: `notes/<basename>.md`

2. **Проверить существующий файл:**
   - Если `notes/<basename>.md` уже есть — спросить пользователя:
     - **Перезаписать** (overwrite)
     - **Дописать в конец** (append)
     - **Отменить** (cancel)
   - Если новый — создать.

3. **Содержимое файла:**

```markdown
# <Краткий заголовок темы — на основе сути ответа>

> Сохранено: YYYY-MM-DD HH:MM
> Источник: переписка с Claude Code

---

<МОЙ ПОСЛЕДНИЙ ОТВЕТ — дословно, в Markdown, с сохранением форматирования>

---

## Заметки (заполняется вручную)

<пусто — место для будущих правок Михаила>
```

   - Заголовок составить **из сути ответа** (1 строка, до 70 символов).
   - Дату взять текущую системную (`date '+%Y-%m-%d %H:%M'`).
   - Дословно сохранить **последнее моё сообщение** в чате (тот ответ, что был непосредственно перед командой `/save-last`).

4. **После сохранения:**
   - Подтвердить: «Сохранено в `notes/<basename>.md`. Открыть в VS Code: cmd+click на ссылке.»
   - Дать markdown-ссылку: `[notes/<basename>.md](notes/<basename>.md)`

## Что НЕ делать

- ❌ Не сохранять, если последний ответ был коротким техническим (1-2 строки) — спросить «уверен, что хочешь сохранить именно этот короткий ответ?»
- ❌ Не сохранять автоматически без команды
- ❌ Не редактировать суть ответа — только дословно

## Примеры

**Пример 1:**
```
Михаил: /save-last methodology-explained
Claude: [сохраняет в notes/methodology-explained.md]
        Сохранено в [notes/methodology-explained.md](notes/methodology-explained.md).
```

**Пример 2 (без имени):**
```
Михаил: /save-last
Claude: Под каким именем сохранить?
Михаил: roles-and-skills
Claude: [сохраняет]
```

**Пример 3 (файл существует):**
```
Михаил: /save-last methodology-explained
Claude: Файл notes/methodology-explained.md уже есть. Перезаписать / дописать / отменить?
Михаил: дописать
Claude: [append с разделителем "---" и новой датой]
```
